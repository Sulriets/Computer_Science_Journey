# Capítulo 02: ¿Cómo se escribe en Markdown?

Creo que este es el momento que todos esperábamos. Tranquilidad, esto no es tan difícil como uno puede llegar a creer. Lo que queremos lograr es un "Aprender haciendo" ✍🏻. En este capítulo no vas a memorizar Markdown. Lo vas a escribir. La mejor forma de aprender este lenguaje es usándolo desde el primer minuto.

## 1. ¿Para qué sirve?

Imagina que estás tomando apuntes para estudiar o haciendo un informe. Sería contradictorio no tener un orden o esquema fijo. 

Partes de tu texto deben destacar, otras no tanto. Algunas pueden entenderse mejor tras una tabla o lista. En palabras simples.. ser **organizado**. Para que tanto el lector como la computadora y los colaboradores estén en la misma página. 

---
## 2. ¿Qué aprenderemos? 

* Encabezados
* Párrafos
* Negrita
* Cursiva
* Tachado
* Código
* Listas
* Citas
* Líneas horizontales
* Enlaces
* Imágenes
* Tablas 
* Bloques de código

Lo sé, parece demasiado. 😨 Pero no te preocupes. Una vez que escribes dos o tres documentos, muchas de estas cosas terminan saliendo de memoria. 😌 

Instrucciones: Markdown lo vas a usar siempre para documentar tu progreso. Por lo que te aconsejo dividir tu pantalla usando `Windows` + `→ (Derecha)` o `← (Izquierda)` de tus teclas de navegación y tener tu editor de código y este documento de Markdown visibles.

Sin más que agregar, vamos a ello.
